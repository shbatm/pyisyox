# Building the docs

The docs are built with [Sphinx](https://www.sphinx-doc.org/) using the
`sphinx_rtd_theme`. They target eisy / Polisy on IoX 6 â€” the public
API exposed by the `pyisyox` package at this version.

## One-time setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r docs/requirements.txt
pip install -e .                  # so autodoc can import pyisyox
```

`aiohttp` is in `docs/requirements.txt` because `pyisyox` imports it at
module load time; without it, `autodoc` cannot introspect the package.

## Build

From the `docs/` directory:

```bash
make html                         # HTML output â†’ docs/_build/html
```

Or directly:

```bash
sphinx-build -b html -n . _build/html
```

Open `docs/_build/html/index.html` in a browser.

## Regenerate the API reference

The contents of `docs/api/` are auto-generated by `sphinx-apidoc` from
the current package layout. Run this whenever modules are added or
moved under `pyisyox/`:

```bash
make apidoc                       # equivalent to:
                                  # rm -rf docs/api
                                  # sphinx-apidoc -f -o docs/api -e -M \
                                  #     -H "API Reference" --tocfile index \
                                  #     pyisyox pyisyox/__main__.py
```

Flags:

- `-f` â€” force overwrite of existing files.
- `-o api` â€” output directory.
- `-e` â€” one file per module.
- `-M` â€” put module docs before submodules.
- `-H` â€” heading for the toctree.
- `--tocfile index` â€” the toctree page is named `index.rst`.
- `pyisyox/__main__.py` is excluded so it doesn't get listed as a
  module entry (it's the CLI, not part of the API).

## Warnings

The build emits ~160 warnings, all in suppressed categories
(`ref.python` duplicates from re-exports at `pyisyox.X`, plus dataclass
`Attributes:` blocks that napoleon documents twice). These are
cosmetic; the build still succeeds. The relevant suppression is in
`conf.py`'s `suppress_warnings`. If you want a fully clean build, the
two remaining warning sources are:

1. **Top-level re-exports** â€” `pyisyox/__init__.py` exposes symbols
   like `Controller` that also live at `pyisyox.controller.Controller`.
   Both get autodoc'd. Suppressing `ref.python` is the simplest fix.
2. **`Raises:` blocks with `/` in them** â€” autodoc tries to interpret
   `Raises: ClientError / HTTPError / AuthError` as a single
   exception reference. Reword those docstrings to separate the
   exceptions onto their own lines if you want the warnings gone.
